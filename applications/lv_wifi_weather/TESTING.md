# 验收场景

以下场景使用 `LVWW_USING_MOCK=y`、800×480 横屏和触摸输入执行。真实硬件接入时应重复
同一组测试，并额外验证 Wi-Fi 驱动、TLS、天气服务和安全存储实现。

## 首页与布局

- 启动后首页、底部三个导航按钮均完整显示，任何对象不超出 800×480。
- 首页无缓存时显示占位信息；联网后显示城市、当地时间、天气及今日摘要。
- 底部导航和 Wi-Fi 操作按钮的有效触控区域不小于 48 px。
- 断网后保留天气数据并显示“离线缓存”，重新连接后立即更新。

## Wi-Fi

- 首次进入 Wi-Fi 页显示 5 个模拟热点，信号、开放/加密状态正确。
- `Coffee-Free` 无需密码即可连接；加密网络拒绝少于 8 字符的密码。
- 输入 `wrongpass` 触发“Wi-Fi 密码错误”，合法密码连接成功并保存账号。
- 重新编辑账号时密码默认隐藏，“显示/隐藏”按钮工作正常。
- 删除账号后列表与自动重连记录同步更新；删除当前连接账号会断开网络。
- 保存 8 个不同账号后，第 9 个账号给出数量上限提示，不覆盖已有账号。
- 重启组件后，KV 数据有效时自动连接最近一次成功网络。

## 城市、时间和天气

- 城市输入不足 2 个字符时不发请求；停止输入 500 ms 后只发起一次最新查询。
- 搜索 `bei`、`shang`、`lon` 和 `new` 能得到对应候选，点击后返回首页并立即请求天气。
- Beijing 使用 UTC+8；London 和 New York 使用模拟后端返回的城市偏移。
- 通过 UTC 临近 00:00 的测试事件验证城市日期跨日与星期计算。
- 连续切换城市时，旧请求的迟到天气不会覆盖新城市结果。

## 错误与生命周期

- `lvww_mock_set_network_available(mock, RT_FALSE)` 触发掉线、天气失败和校时失败提示。
- `lvww_mock_set_storage_failure(mock, RT_TRUE)` 后新增/删除账号与换城市均显示存储失败。
- 连续快速点击扫描不会创建并行扫描；队列满时 `lvww_post_event()` 返回错误而不阻塞 UI。
- 销毁前解除后端绑定，活动请求收到取消，后端不再向已释放的 UI 上下文投递事件。

## 已执行的构建检查

- ARM GNU GCC，C99，`-Wall -Wextra -Werror -fsyntax-only`。
- RT-Thread 4.1.1 真实头文件。
- LVGL 官方 v8.3.11 真实头文件。
- `LV_USE_USER_DATA=0`，确认组件不依赖对象 user data。
- `LV_FONT_SIMSUN_16_CJK=1`、`LV_FONT_MONTSERRAT_48=1`，确认中文与大号字体配置可编译。
